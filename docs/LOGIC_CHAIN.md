## 链路一：产品入库

**角色：** 产品经理 / 管理员  
**入口：** `/admin` → 产品管理

```
┌─────────────────────────────────────────────────────────────────┐
│                         产品入库链路                              │
└──────────────────────────────┬──────────────────────────────────┘
                               │
         ┌─────────────────────┼─────────────────────────┐
         ▼                     ▼                         ▼
   ┌───────────┐        ┌────────────┐           ┌────────────────┐
   │ 手动创建   │        │ Excel 导入  │           │  1688 链接录入  │
   │ 单个产品   │        │ 批量导入    │           │  (source_url)  │
   └─────┬─────┘        └─────┬──────┘           └───────┬────────┘
         │                   │                           │
         └─────────┬─────────┘───────────────────────────┘
                   ▼
         ┌─────────────────────┐
         │   products 表写入    │
         │  sku / name / image  │
         │  price(CNY含税价)    │
         │  moq / volume / weight│
         │  status: active      │
         └──────────┬──────────┘
                    │
         ┌──────────▼──────────┐
         │    配置加价规则        │
         │                     │
         │  ① 全局加价          │   → app_settings 表
         │     所有产品 +X%      │
         │                     │
         │  ② 分类加价          │   → categories.markup_value
         │     某分类产品 +Y%    │
         │                     │
         │  ③ 单品覆盖          │   → product_markups 表
         │     指定产品 = Z 元   │
         └──────────┬──────────┘
                    │
                    ▼
         ┌─────────────────────┐
         │   前台展示价 = 含税价  │
         │   × 汇率(CNY→USD)   │

         │                     │

         └─────────────────────┘

**数据流：**

| 步骤 | 写入表 | 关键字段 |
|------|--------|---------|
| 创建产品 | `products` | id, sku, name, price, moq, volume_m3, weight_kg, status |
| 规格变体 | `product_specs` | product_id, label, price, stock |
| 分类关联 | `product_categories` | product_id, category_id, is_primary |
| 加价规则 | `product_markups` | product_id, markup_value, markup_type |
| 导入记录 | `import_batches` | source_file, status, total_rows, success_rows |

```

customer_identities → session / email / whatsapp 三条
conversations     → channel="whatsapp", status="open"

customer_access_tokens → token, expires_at=30天后
email_send_records → 待发送记录（SMTP未配置时标记为pending）
```

```

## 状态汇总

### 报价单状态

| 状态 | 含义 | 下一步 |
|------|------|--------|
| 新询价 | 客户刚提交，待跟进 | 联系客户 → 跟进中 |
| 跟进中 | 正在沟通，未发报价 | 发送报价 → 已报价 |
| 已报价 | PDF/文字报价已发送 | 谈判跟进 → 已成交 / 已关闭 |
| 已成交 | 客户确认下单 | 归档 |
| 已关闭 | 客户放弃或长期无回应 | 归档 |

### 客户状态

| 状态 | 含义 | 对应场景 |
|------|------|---------|
| 潜在客户 | 刚询价，未深度沟通 | 新询价阶段 |
| 跟进中 | 正在沟通，未成交 | 跟进中/已报价阶段 |
| 活跃 | 成交过，保持联系 | 已成交后 |
| 失效 | 长期无回应或明确不买 | 已关闭后 |

### 会话状态

| 状态 | 含义 |
|------|------|
| open | 进行中，业务员可收发消息 |
| closed | 已关闭，不再接收新消息 |
