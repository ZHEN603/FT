# 外贸获客与报价成交平台

网站搭建需求文档 / PRD V1.0

项目定位：一个后台系统，新增“社媒运营中心”和“沟通中心”两个一级菜单，打通产品、内容、线索、聊天、询盘、报价与成交。

| 文档项 | 内容 |
| --- | --- |
| 适用对象 | 老板 / 产品经理 / UI 设计 / 前端开发 / 后端开发 / 测试 |
| 项目范围 | 前台产品网站 + 后台管理系统 + 社媒运营中心 + 沟通中心 |
| 一期目标 | 跑通从产品展示、社媒获客、客户沟通、自动翻译、询价、报价到跟进成交的核心闭环 |
| 文档日期 | 2026-05-26 |

## 1. 项目背景

当前项目面向外贸团队，原有系统已经包含产品管理、询盘管理、报价单管理、客户管理、跟进记录、供应商管理、加价管理、装柜计算和海运估价等能力。新的建设方向是在此基础上加入海外社媒获客和多渠道聚合沟通能力，让业务员在同一个后台完成获客、沟通、报价和成交。

本项目不是单独做一个商城，也不是单独做一个客服工具，而是建设一个面向外贸业务的“获客 + 沟通 + 报价 + 成交”一体化平台。

## 2. 产品定位

产品定位：面向外贸企业的 AI 获客、聚合沟通与智能报价平台。

- 帮助外贸企业统一管理产品、供应商、客户、询盘、报价和跟进记录。
- 帮助运营人员从产品库快速生成海外社媒内容，并一键发布到 Instagram、Facebook、TikTok。
- 帮助业务员在后台统一接收 WhatsApp、Instagram、Facebook、TikTok 等渠道消息。
- 支持中文输入自动翻译成英文发送，海外客户英文回复自动翻译成中文展示。
- 将社媒线索和聊天内容沉淀为客户、询盘、报价单和跟进记录。

## 3. 总体建设原则

| 原则 | 说明 |
| --- | --- |
| 一个后台 | 所有功能接入现有外贸后台，不拆成多个独立系统。用户登录一个后台即可完成工作。 |
| 两个新增一级菜单 | 新增“社媒运营中心”和“沟通中心”，分别负责获客和转化。 |
| 前台分模块 | 产品、社媒、聊天、客户、询盘、报价等页面清晰分开，避免一个页面塞太多功能。 |
| 底层数据打通 | 产品、客户、消息、内容、询盘、报价单之间可互相关联，形成完整业务闭环。 |
| 一期先跑通闭环 | 优先实现核心业务链路，复杂自动化和高级数据分析可放到二期。 |

## 4. 目标用户与角色

| 角色 | 主要工作 | 核心页面 |
| --- | --- | --- |
| 老板 / 管理者 | 查看询盘、报价、成交、社媒获客和业务员效率 | 仪表盘、分析管理、客户管理、报价单管理 |
| 产品经理 / 管理员 | 维护产品体系、菜单权限、业务流程和系统配置 | 产品管理、系统设置、操作日志 |
| 外贸运营 | 维护账号矩阵、生成内容、发布视频图文、查看发布数据 | 社媒运营中心、素材库、发布记录 |
| 外贸业务员 | 回复客户消息、自动翻译、跟进询盘、发送报价单 | 沟通中心、客户管理、询盘管理、报价单管理 |
| 采购 / 供应链 | 维护供应商、1688 货源、成本价、加价规则 | 供应商管理、加价管理、产品管理 |

## 5. 总体业务流程

核心流程：供应商/1688 选品 → 产品库 → 加价规则 → 前台展示 / 社媒发布 → 客户浏览或互动 → 沟通中心聊天 → 转客户/询盘 → 生成报价单 → WhatsApp 跟进 → 成交或继续跟进。

| 步骤 | 说明 | 关联模块 |
| --- | --- | --- |
| 1. 选品入库 | 从供应商或 1688 采集产品，维护 SKU、图片、规格、MOQ、成本价、分类。 | 产品管理、供应商管理 |
| 2. 设置销售价 | 按分类、产品或规则设置加价比例，生成面向海外客户的报价基础价。 | 加价管理 |
| 3. 前台展示 | 客户通过网站搜索、筛选和查看产品，加入询盘或集装箱。 | 前台产品网站 |
| 4. 社媒获客 | 运营选择产品，生成英文文案、图片或视频，一键发布到海外平台。 | 社媒运营中心 |
| 5. 消息聚合 | WhatsApp、社媒私信和评论进入后台，业务员统一处理。 | 沟通中心 |
| 6. 转化询盘 | 客户有意向后，一键转客户、转询盘，关联产品和聊天记录。 | 客户管理、询盘管理 |
| 7. 报价成交 | 业务员生成报价单、发送 PDF、继续跟进，最终转成交订单。 | 报价单管理、跟进记录 |

## 6. 网站前台需求

### 6.1 前台定位

前台是面向海外客户的 B2B 产品目录网站，核心目标不是直接在线支付，而是让客户快速找到产品、组合采购需求、提交询价，并通过 WhatsApp 与业务员建立联系。

### 6.2 前台核心页面

| 页面 | 需求说明 |
| --- | --- |
| 首页 | 展示品牌、核心优势、主推产品分类、精选产品、快速入口、询盘入口。 |
| 产品列表页 | 支持分类、材质、尺寸、表面处理、库存、可定制、价格排序、列表/网格切换。 |
| 产品详情页 | 展示产品图片、型号、规格、材质、MOQ、价格、包装信息、加入询盘/集装箱。 |
| 分类页 | 按塑料衣架、木质衣架、金属衣架、植绒衣架、裤架/裙架、儿童衣架、配件等展示。 |
| 我的询盘 / 集装箱 | 显示已选产品、数量、体积、重量、柜型利用率、费用估算和提交询价表单。 |
| 提交询价弹窗 | 客户填写姓名、国家/地区、WhatsApp、邮箱、公司名、备注，同意通过 WhatsApp 沟通。 |

### 6.3 前台关键能力

- 产品搜索：支持产品名称、型号、SKU、关键词搜索。
- 加入询盘：客户可将多个产品加入询盘清单。
- 装柜计算：按产品包装尺寸、数量、重量计算 20GP、40GP、40HQ、45HQ 装柜情况。
- 海运估价：选择起运港、目的港后，显示海运费、港口杂费、文件费、报关费、保险费等参考费用。
- WhatsApp 引导：提交询价后提示客户通过 WhatsApp 联系专属顾问。
- 多语言：一期至少支持中文后台、英文前台；后续支持多语言站点。

## 8. 新增一级菜单：社媒运营中心

### 8.1 模块定位

社媒运营中心负责“发出去”，即把产品内容发布到 Instagram、Facebook、TikTok 等海外平台，提升曝光并获取评论、私信和询盘线索。

### 8.2 子菜单设计

| 子菜单 | 功能说明 | 一期优先级 |
| --- | --- | --- |
| 账号矩阵 | 绑定和管理 Instagram、Facebook Page、TikTok 账号，支持按品牌、国家、业务员、产品线分组。 | P0 |
| 内容发布 | 支持上传视频/图片，填写标题、正文、标签，选择多个平台和账号，一键发布或定时发布。 | P0 |
| 素材库 | 沉淀产品图片、视频、文案、标签和模板，可从产品库直接选素材。 | P0 |
| 发布记录 | 记录每条内容发布平台、账号、时间、状态、失败原因和重试记录。 | P0 |
| 评论私信 | 聚合社媒评论和私信，支持查看、回复、翻译和转客户/询盘。 | P1 |
| 运营数据 | 展示发布量、曝光、互动、私信、线索、转询盘、转报价等数据。 | P1 |
| 账号环境/节点配置 | 管理账号访问环境、海外节点、代理配置和账号风控隔离。建议使用合规表达。 | P1 |

### 8.3 内容发布流程

- 运营进入产品库，选择要推广的产品或产品组合。
- 系统自动带出产品图片、视频、SKU、卖点、价格区间、MOQ 和询盘链接。
- AI 生成英文标题、正文、标签，可手动编辑。
- 选择发布平台：Instagram、Facebook、TikTok。
- 选择账号：可选择一个或多个账号。
- 选择发布方式：立即发布或定时发布。
- 发布后生成发布记录，并回传成功、失败、审核中等状态。

### 8.4 关键字段

| 对象 | 字段 |
| --- | --- |
| 社媒账号 | 平台、账号名称、账号头像、授权状态、所属业务员、所属品牌、国家地区、账号分组、最近同步时间。 |
| 发布内容 | 标题、正文、标签、素材、关联产品、发布平台、发布账号、发布时间、发布状态、失败原因。 |
| 素材 | 素材类型、文件、关联产品、适用平台、语言、标签、创建人、使用次数。 |

## 9. 新增一级菜单：沟通中心

### 9.1 模块定位

沟通中心负责“聊回来”，即把 WhatsApp、Instagram、Facebook、TikTok 等渠道的客户消息集中到后台，业务员用中文处理，系统自动翻译并沉淀跟进记录。

### 9.2 子菜单设计

| 子菜单 | 功能说明 | 一期优先级 |
| --- | --- | --- |
| 全部会话 | 展示所有渠道会话，支持按渠道、业务员、客户状态、未读、标签筛选。 | P0 |
| WhatsApp 会话 | 业务员在后台直接处理 WhatsApp 客户消息，发送文本、图片、报价单 PDF 和产品卡片。 | P0 |
| Instagram 私信 | 聚合 Instagram 私信和评论线索，支持回复、翻译和转客户。 | P1 |
| Facebook Messenger | 聚合 Facebook Page 消息和评论，支持回复、翻译和转客户。 | P1 |
| TikTok 私信/评论 | 聚合 TikTok 评论和私信能力；具体能力需以平台 API 权限为准。 | P1 |
| 自动翻译 | 中文输入自动翻译成英文发送，英文消息自动翻译成中文展示。 | P0 |
| 快捷话术 | 维护 MOQ、价格、交期、付款方式、物流、样品等常用话术。 | P0 |
| 会话分配 | 按客户国家、产品分类、账号来源或业务员规则分配会话。 | P1 |

### 9.3 聊天页面布局

| 区域 | 内容 |

| 左侧会话列表 | 客户名称、渠道图标、最近消息、未读数、国家地区、负责人、客户状态、关联询盘/报价。 |
| 中间聊天窗口 | 消息时间线、原文、译文、输入框、附件、产品卡片、报价单、快捷话术、AI 回复建议。 |

### 9.4 翻译发送逻辑

| 场景 | 系统逻辑 |
| --- | --- |
| 业务员输入中文 | 点击发送后，系统自动翻译成英文；可设置为先预览再发送或直接发送。 |
| 客户回复英文 | 系统保留英文原文，同时展示中文译文。 |

## 10. 现有核心模块需求

### 10.1 产品管理

- 维护产品分类：塑料衣架、木质衣架、金属衣架、植绒衣架、裤架/裙架、儿童衣架、衣架配件、展示架/挂钩、套装组合、定制服务。
- 维护产品信息：SKU、名称、型号、材质、尺寸、颜色、表面处理、MOQ、价格、库存、包装尺寸、重量、图片、上下架状态。
- 支持多币种价格展示，例如 CNY、USD、EUR、GBP、JPY、AUD、CAD、HKD、INR。
- 支持产品数据统计：浏览次数、加入询盘、WhatsApp 点击、报价次数、收藏次数。

### 10.2 加价管理

- 支持 1688 商品成本价导入或采集。
- 支持按分类、单品、固定金额、百分比等方式设置加价。
- 支持批量应用规则、批量改价、预览结果和同步到前台。
- 保留原价、当前加价、加价后价格和加价规则来源。

### 10.3 询盘管理

- 记录客户提交的产品需求、联系方式、来源渠道、关联产品和装柜信息。
- 支持询盘状态：新询盘、待报价、已报价、跟进中、已成交、已失效。
- 支持从社媒评论/私信、WhatsApp 会话一键转询盘。
- 支持指定负责人、设置下次跟进时间和生成跟进记录。

### 10.4 报价单管理

- 基于客户、产品、数量、装柜信息、海运费用生成报价单。
- 报价单包含客户信息、产品明细、费用信息、集装箱信息、报价总额、供应商匹配、跟进记录。
- 支持生成 PDF、通过 WhatsApp 发送、标记已报价、转为成交订单。
- 支持报价单状态：待报价、已报价、跟进中、已成交、已失效。

### 10.5 客户管理

- 维护客户公司、联系人、国家地区、WhatsApp、邮箱、客户分组、客户状态、负责人。
- 统计累计报价单、成交报价单、累计成交金额、首次询盘时间、最后跟进时间。
- 查看最近报价单、最近跟进记录和备注信息。
- 支持从询盘、聊天、社媒互动一键创建客户。

### 10.6 供应商管理

- 管理供应商名称、主营产品、所在地区、经营模式、1688 店铺、合作状态、响应速度、回头率、产品质量评分。
- 支持查看供应商相关产品、最近报价单、最近跟进和合作数据。
- 支持从报价单详情中匹配供应商，并记录产品与供应商对应关系。

## 13. 技术与接口注意事项

- WhatsApp 建议优先采用 WhatsApp Business API 或官方 Cloud API，避免依赖不稳定的个人号扫码方案。
- Instagram 和 Facebook 建议通过 Meta Graph API、Instagram Graph API、Page API 接入，发布和消息权限通常需要平台审核。
- TikTok 发布能力建议评估官方 Content Posting API；评论、私信能力需以账号类型和 API 权限为准。
- 自动翻译需保存原文和译文，避免客户纠纷时无法追溯。
- 账号环境/节点配置需采用合规表达，例如海外节点、合规代理、账号网络环境管理，不建议在产品文案中直接写“翻墙”。
- 发布失败、授权过期、账号异常、平台限流等情况需要有明确状态提示和重试机制。

## 14. 关键状态设计

| 对象 | 状态建议 |
| --- | --- |
| 询盘 | 新询盘、待报价、已报价、跟进中、已成交、已失效。 |

| 社媒账号 | 未授权、已授权、授权过期、异常、暂停使用。 |

## 16. 建议开发阶段

| 阶段 | 建设内容 | 目标 |
| --- | --- | --- |
| 第一阶段：业务闭环 | 前台产品、询盘、集装箱、后台产品/客户/询盘/报价/WhatsApp 会话/自动翻译。 | 先让业务员能在后台完成询价到报价跟进。 |
| 第二阶段：社媒获客 | 账号矩阵、素材库、社媒一键发布、发布记录、基础数据统计。 | 让产品内容能批量触达海外客户。 |
| 第三阶段：全渠道聚合 | Instagram、Facebook、TikTok 评论私信聚合，转客户、转询盘、会话分配。 | 把社媒线索统一沉淀到客户和报价体系。 |
| 第四阶段：AI 自动化 | AI 文案、AI 回复建议、客户意向评分、自动生成跟进记录、成交预测。 | 提升业务员效率和管理层决策能力。 |

## 17. 验收标准

- 客户可在前台完成产品浏览、加入询盘、装柜计算、海运估价和提交询价。
- 后台可看到客户询盘，并能关联客户、产品、报价单和跟进记录。
- 业务员可在沟通中心处理 WhatsApp 消息，中文自动翻译成英文发送，英文回复自动翻译成中文展示。
- 运营可从产品库选择产品生成社媒内容，并发布到至少一个海外社媒平台。
- 社媒线索或聊天消息可一键转客户或询盘。
- 报价单可生成 PDF，并通过 WhatsApp 发送给客户。
- 管理者可在仪表盘看到产品、询盘、报价、客户、社媒和沟通的核心数据。

## 18. 产品一句话

我们不是做两个独立工具，而是在现有外贸报价后台里新增两个业务模块：社媒运营中心负责获客，沟通中心负责转化，最终打通产品、内容、线索、聊天、询盘、报价和成交。
